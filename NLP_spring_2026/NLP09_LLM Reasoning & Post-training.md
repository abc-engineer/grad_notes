## 9강 LLM Reasoning & Post-training

### Reasoning: Overview
- 추론(대략적 정의): 언어 모델로 텍스트를 생성하거나 분석하여 연역적, 귀납적, 상식적 또는 논리적 추론을 수행하는 것    
- 연역적 추론: 일반 원리나 전제로부터 구체적인 결론을 도출하는 것    
	- 예: “모든 인간은 죽는다” + “소크라테스는 인간이다” $\Rightarrow$ “소크라테스는 죽는다”    
- 귀납적 추론: 구체적인 관찰을 바탕으로 일반화를 만드는 것    
	- 예: “해는 매일 동쪽에서 떴다” $\Rightarrow$ “해는 내일 동쪽에서 뜰 것이다”    
- 상식적 추론: 세계 지식이나 상식적 이해에 의존하여 예측하거나 질문에 답하는 것    
	- 예: “공을 떨어뜨리면 어떻게 될까?” $\Rightarrow$ “공은 떨어질 것이다”    
- 수학적/논리적 추론: 올바른 답에 도달하기 위해 특정 규칙이나 절차를 따르는 것    
	- 예: “사과 3개가 $6달러라면, 사과 5개는 얼마인가?” $\Rightarrow$ “$10달러”

- reasoning: 추론이라고 생각 하면 됨. 요즘 아주 남용 되는 단어 중 하나.

### Frontier LLMs Are Reasoning Models
- 하나의 통합 시스템  
	- GPT-5는 대부분의 질문에 답하는 스마트하고 효율적인 모델, 더 어려운 문제를 위한 **deeper reasoning model 더 깊은 추론 모델(GPT-5 thinking)**, 그리고 대화 유형, 복잡성, 도구 필요 여부, 사용자의 명시적 의도(예: 프롬프트에 “이것에 대해 깊이 생각해 봐”라고 말한 경우)에 따라 어떤 모델을 사용할지 빠르게 결정하는 실시간 라우터를 갖춘 통합 시스템이다. 라우터는 사용자가 모델을 전환하는 경우, 응답 선호도, 측정된 정답률을 포함한 실제 신호를 바탕으로 지속적으로 학습하며 시간이 지남에 따라 개선된다. 사용량 제한에 도달하면 각 모델의 미니 버전이 나머지 요청을 처리한다. 가까운 미래에는 이러한 기능을 하나의 모델로 통합할 계획이다.


### Chain-of-thought (CoT) Prompting
- 사고의 사슬(CoT): 모델이 복잡한 문제를 **단계별 추론 과정으로 분해**하는 것    
- 질문이나 과제에 직접 답을 제공하는 대신, 모델이 자신의 추론 또는 사고 과정을 논리적 순서로 설명하도록 프롬프트를 받는다

- CoT를 최초 소개한 논문은 구글 논문이었음
- 바로 답을 묻는 것이 아니라 추론 과정을 시퀀스로 프롬프트가 되도록 질문하는 방법임.


### Standard Prompting vs. CoT Prompting
#### 표준 프롬프팅
- 모델 입력
	- 질문: Roger는 테니스 공 5개를 가지고 있다. 그는 테니스 공 3개가 들어 있는 캔 2개를 산다. 이제 그는 테니스 공을 몇 개 가지고 있는가?
	- 답: 정답은 11이다.
	- 질문: 구내식당에는 사과 23개가 있었다. 점심을 만들기 위해 20개를 사용하고 6개를 더 샀다면, 이제 사과는 몇 개인가?
- 모델 출력
	- 답: 정답은 27이다. ❌  
#### 사고의 사슬 프롬프팅
- 모델 입력
	- 질문: Roger는 테니스 공 5개를 가지고 있다. 그는 테니스 공 3개가 들어 있는 캔 2개를 산다. 이제 그는 테니스 공을 몇 개 가지고 있는가?
	- 답: Roger는 처음에 테니스 공 5개를 가지고 있었다. 테니스 공 3개가 들어 있는 캔 2개는 테니스 공 6개이다. $5+6=11$. 정답은 11이다.
	- 질문: 구내식당에는 사과 23개가 있었다. 점심을 만들기 위해 20개를 사용하고 6개를 더 샀다면, 이제 사과는 몇 개인가?
- 모델 출력
	- 답: 구내식당에는 처음에 사과 23개가 있었다. 점심을 만들기 위해 20개를 사용했으므로 $23-20=3$개가 남았다. 그런 다음 사과 6개를 더 샀으므로 $3+6=9$개가 있다. 정답은 9이다. ✅

- CoT로 질문하면 답변도 논리적으로 중간 과정도 리턴 해줌.
- 생각의 과정을 함께 생성하게 하는 프롬프팅 임.

### Self-consistency CoT
직관: 여러 가지 서로 다른 사고 방식이 같은 답으로 이어진다면, 최종 답이 맞다는 확신이 더 커진다.


- CoT로 답한다고 무조건 맞는건 아님.
- CoT를 보완하는 기법중 하나가 Self-consistency CoT
	- 답을 낼때 일관적 사고 과정이 있으면 일관된 답을 낼것이다 (consistency)
	- 같은 인풋에 샘플링을 할때 여러 샘플을 얻고 여러개의 추론 패스를 통한 답 여러개를 얻을 수 있음.
	- 이 여러개의 답을 비교하면 더 정확한 답을 얻을 수 있음
	- 수학적으로 marginalize 한다고 표현
		- 필요한 부분만 남겨서 계산
- **신뢰도 확보를 위한 LLM의 범용적인 방법**

	- 단순히 시간을 길게 하는게 아니라 모델이 추론을 스스로 검증하고 오답을 고치게 하는게 포인트

	- 여러 추론 경로를 동시에 탐색한다

	- 처음부터 완벽한 답이 아니라 반복을 통해 개선선

- 이전에는 학습에 많은 리소스를 투여해서 답의 품질을 높혔지만 이제는 test time에 더 많은 비용을 투여해서 답변의 품질을 높임 → Test-time Scaling이라고 함.


### Reinforcement Learning with Verifiable Rewards (RLVR)
- **OpenAI의 o1과 DeepSeek-R1의 주요 사후 훈련 방식**    
- RLVR:    
	- 강화학습을 사용해 정책 모델(LLM)을 파인튜닝한다    
	- LLM은 생성한 응답이 검증 가능하게 올바를 때 보상을 받는다

- 모델이 답을 내게 하고 그 답이 좋으면 보상을 하도록 설계.
- $s_i$ : 프롬프트
- $\theta$: 파라미터
- $a_i$: 액션
- $r_i$: 보상 reward
	- 맞으면 r만큼 리워드 틀리면 0의 리워드
	- 이걸 하는게 verifier
- 리워드를 높이는 방향으로 학습 (리워드를 최대화 하도록 파라미터 조정 그래서 목적함수에 + 사용)
	- $\theta_{t+1}=\theta_t+\alpha\nabla_{\theta}J(\pi_{\theta})$

- 정답여부를 스스로 확인 할 수 있는 문제에서 스스로 강화 학습.
- 어떤 답을 해야 하는지 판단하는 모델이 policy model
- 기계학습에서는 목적함수를 정하고 그라디언트를 최소화, RLVR에서는 리워드를 정의하고 리워드를 최대화


### Why RLVR?
- 데이터 확장성
	- 지도학습은 인간 주석자가 완전한 정답 응답을 만들어야 한다
	- RLVR은 응답을 채점하기 위한 자동 verifiers (검증기)만 필요로 한다
- 모방 vs. 최적화
	- 지도학습은 모델이 인간의 추론 단계를 모방하도록 강제하는데, 이는 차선일 수 있다
	- RLVR은 올바른 최종 답을 직접 최적화하여, 모델이 자신만의 효율적인 추론 경로를 발견할 수 있게 한다
- 분포 불일치
	- 인간이 만든 데이터로 수행되는 지도학습은 모델 자신의 분포와 불일치를 일으킬 수 있다
	- RLVR은 모델이 직접 생성한 시퀀스로부터 학습하므로, 추론 분포와 일치한다




### RLVR Setup
- LLM(정책 모델)을 사용해 응답을 생성한다
- 생성된 응답에 보상을 할당한다
- 기대 보상을 최대화한다  

$$  
\max_{\theta}\mathbb{E}_{y \sim p_{\theta}(\cdot \mid x)}\left[r_{\mathrm{RLVR}}(x,y)\right]  
$$

- $p_{\theta}(\cdot \mid x)$: LLM 출력 확률
- $r_{\mathrm{RLVR}}(x,y)$: x 인풋, y 아웃풋의 RLVR의 보상(이진)






### Optimization with Reinforcement Learning (RL)
- 왜 강화학습인가:
	- 사용 가능한 지도 데이터가 없다
	- 보상에 의해 안내되며 모델이 새로운 가능성(생성)을 탐색하도록 장려한다
- 최적화: policy gradient methods 정책 그래디언트 방법
	- 기대 보상을 증가시키는 방향으로 파라미터를 조정하여 정책(LLM)을 최적화한다
	- 가장 기본적인 방법
		- 리워드를 최대화 하자
- REINFORCE(가장 단순한 정책 그래디언트 방법):  

$$  
\theta \leftarrow \theta + \alpha \nabla_{\theta}\log \pi_{\theta}(a \mid s)R  
$$

- $\alpha$: 스텝 크기
- $\pi_{\theta}$: 정책 모델(LLM)
- $a$: 행동(응답 생성)
- $s$: 상태(사용자 프롬프트 + 대화 기록)
- $R$: 누적 보상






### Overview: Proximal Policy Optimization (PPO)
- 안정성과 효율성을 향상시키는 **더 발전된 정책 그래디언트 방법**    
- Clipped mechanism 클리핑 메커니즘: PPO는 클리핑된 대리 목적 함수를 사용하여 정책 업데이트가 너무 커지지 않도록 하며, 이는 안정성을 유지하는 데 도움이 된다    
- 어드밴티지 추정: PPO는 비평가 모델과 함께 일반화 어드밴티지 추정(GAE)을 사용하여 어드밴티지 추정의 분산을 줄이고, 학습 효율성을 향상시킨다

- policy gradient method 보다 더 고도화 된 방법 중 하나.


### Overview: Group Relative Policy Optimization (GRPO)
- 비평가 모델을 사용하지 않는 PPO의 변형    
- 어드밴티지 추정: 각 질문에 대해 GRPO는 출력 그룹을 샘플링하고, 비교 보상을 사용하여 어드밴티지를 추정한다

- 최근에 PPO보다 더 많이 쓰이는 방법
- $o$: output
- $r$: 리워드
- $v$: value model 보상이 얼마나 적정한가를 판단 하는 모델 (또다른 llm모델임)
	- 기준점이 되어서 중요한 업데이트는 리워드를 더 주게 하는데 그것을 판단하는 모델
- $A$: advetage
- $KL$: 기존 언어 모델에서 너무 벗어나지 않도록 하는 역할
	- 리워드 높이는데 집중하면 기존 모델의 능력을 잃어버리는 경우가 있음.
- Generalized Advantage Estimation (GAE): 베이스라인 대비 밸류 모델이 얼마나 어드벤티지를 줄지 판단.
- reference model: 업데이트 되지 않고 원래 모델에서 너무 벗어나지 않도록 기준점을 잡아주는 역할을 하는 모델
- PPO는 안정적이고 효율적이나 밸류모델이 있어 다른 모델을 학습하고 설계해야 하는 단점이 있고, 밸류 모델이 잘못 되면 PPO 전체가 많이 흔들림.
- GRPO는 딥시크 R1에서 사용되어서 인기가 많아짐
	- 밸류모델을 삭제하고 여러개의 output에 각각 리워드를 계산 함.
	- 그 후 리워드들을 서로 비교함.
	- 어떤 답변이 더 좋았는지 확인
	- 별도의 밸류모델을 기준으로 비교하는게 아니라 서로 답변을 비교
	- 단점
		- PPO에서 밸류모델이 좋은 경우 GRPO보다 더 좋은 결과가 나옴.
		- 하지만 GRPO는 비용이 적지만 PPO를 능가하는 경우도 많이 나와서 요즘 많이 사용.
- group computation: 구현 방법은 무수히 많이 있음. 그냥 프레임 워크 측면에서 그룹의 리워드들을 비교 계산하는 역할을 한다는 것으로 이해하고 실무에서는 모델별로 판단해서 사용.

### The Evolution of GPT Models: ChatGPT
- GPT-1: 디코더 전용 Transformer 사전훈련

	- GPT는 단순 언어 모델 
	- 챗지피티는 언어 모델 얼라이먼트 


### Overview: Language Model Alignment
- 얼라이먼트는 언어 모델의 행동이 일반적인 과제/응용에서 **인간의 가치와 의도에 맞게 정렬되도록 한다** 
- “HHH” 기준(Askell et al. 2021):    
	- Helpful: 사용자가 요청한 작업을 효율적으로 수행한다    
	- Honest: 정확한 정보를 제공하고 불확실성을 표현한다    
	- Harmless: 공격적/차별적/편향된 출력을 피한다

- 단순히 문장을 만드는것이 아니라 사용자의 목적에 맞는 출력을 줄 수 있게 하는 것.
- 할루시네이션이 대표적인 어니스트 위반 사항
- 얼라이먼트는 이 HHH 세가지 기준의 균형을 맞추면서 사용자 요구를 맞추는 것.
- 랭귀지 모델은 그냥 답을 주면 됨. 랭귀지 모델 얼라이먼트는 그냥 답변 + HHH


### Language Model Alignment: Post-training
- 사전훈련된 언어 모델은 정렬되어 있지 않다    
- 목적 함수 불일치    
	- 사전훈련은 문장에서 다음 단어를 예측하는 것이다    
	- 인간의 의도/가치를 이해하는 것을 포함하지 않는다    
- 훈련 데이터 편향    
	- 인터넷의 텍스트는 편향적이거나 유해하거나 오해를 불러일으키는 정보를 포함할 수 있다    
	- LM은 훈련 데이터의 좋은 행동과 나쁜 행동을 구별하지 않는다    
- 과도한 일반화 문제    
	- LM의 일반화는 특정 맥락에서 부적절한 출력으로 이어질 수 있다    
	- 의도된 윤리/정직성 기준과 일치하지 않을 수 있다

- 프리 트레이닝과 상반되게 얼라인먼트는 포스트 트레이닝이라고 함.
- 사전 훈련에서는 다음단어만 예측하게 훈련되었는데 인간의 의도와 가치를 이해하는 답변을 내놔야 함.


### Language Model Alignment Techniques (chatGPT3.5 기준)
- Instruction Tuning (지시 튜닝닝)
- 인간 피드백 기반 강화학습(RLHF)
- 얼라이먼트 하는 방법
	- 3가지 스텝을 거침
		- 1단계: instruct turnning - 폴리시 모델 학습
			- 지시를 했을때 지시에 맞는 답변을 하도록 훈련
			- 모범 질문 답안을 예시로 학습
			- 지시와 답변 쌍을 demonstrates data라고 함.
			- 그 다음 supervised learning
			- SFT: supervised fine turning
				- 이 단계가 없으면 모델이 지시 이행을 못함.
		- 2단계: reward model 학습
			- instruct 모델만으로 답변이 항상 최고라고 할 수 없음
			- 사람들이 원하는 답변을 기반으로 리워드 모델을 학습
			- 하나의 프롬프트에 대해 모델이 여러개의 답변을 내놓고 그 답에 사람이 직접 순위를 메김. → 사람들의 선호를 수집하고 학습하는 과정 
			- 단순히 답이 맞는지 틀린지만 하는것이 아니라 리워드 모델은 사람이 좋아할 답인지까지 학습함.
			- 사람의 선호를 기반으로 리워드를 학습
		- 3단계: Reinforcement Learning from Human Feedback(RNHF)
			- PPO 같은 강화 학습 모델을 사용 (이때는 GRPO 없었음 후에 딥시크에서 출시)
			- 그결과 $r_k$가 계산 됨.
			- 사람에게 맞는 답을 찾는 과정으로 학습되기 때문에 휴먼 피드백이라 함.




### Overview: Instruction Tuning
- 다양한 과제 집합을 사용해 LM을 훈련한다    
	- 각 과제는 지시문과 원하는 출력 예시로 구성된다    
	- 목표는 모델이 특정 지시, 즉 인간의 의도를 효과적으로 따르도록 가르치는 것이다    
- 결과 모델은 문맥 내 시연 없이도 다양한 과제를 제로샷으로 수행할 수 있다    
	- instruction 학습과정을 거쳐야 이렇게 제로샷도 가능해짐.
- instruction (지시문)은 채팅 형식일 수도 있으며, 이는 LM을 챗봇으로 튜닝하는 것이다

- instruct가 붙어 있는게 post trained된 모델

### Overview: RLHF
- 인간 피드백 수집    
	- 같은 프롬프트에 대해 모델을 사용해 여러 응답을 생성한다    
	- 인간 평가자는 유용성/정직성/안전성 등을 기준으로 모델의 응답 순위를 매긴다    
- 보상 모델 훈련    
	- 보상 모델은 인간 피드백 데이터로 훈련되어 응답의 품질을 예측한다    
	- 더 높은 보상 = 인간 평가자가 더 선호함    
- 정책 최적화    
	- 강화학습 알고리즘을 사용해 보상 모델이 예측한 보상을 최대화하도록 LM을 추가로 훈련한다  
	- 모델이 인간 선호와 더 잘 정렬된 출력을 생성하도록 장려한다




### Instruction Tuning: Introduction
- 설정: 다양한 과제에 대해 과제별 지시문으로 LLM을 파인튜닝한다    
- 목표: LLM이 사용자 프롬프트를 더 잘 이해하고, 광범위한 미지의 과제에 제로샷으로 일반화할 수 있게 한다

- 학습되지 않은 질문이 들어와도 일반화 가능하도록 하는게 목적


### Instruction Tuning: Method
- 입력: 과제 설명    
	- 질문에 답해라, 분석해라 , 계산해라 같은 것들.
- 출력: 과제에 대한 기대 응답 또는 해결책    
- 프롬프트가 주어졌을 때 응답 토큰을 생성하도록 LLM을 훈련한다
	- 인간이 정한 정답과 유사하게 답변하도록 오토리그레션 

$$  
\min_{\theta} -\log p_{\theta}(y \mid x)  
$$

- $y$: 응답    
- $x$: 프롬프트


### Instruction Tuning vs. Other Paradigms
- (A)과제별 파인튜닝은 여러 과제에 걸친 일반화를 가능하게 하지 않는다    
- (B) 프롬프트, In-context learning 
	- 문맥 내 학습은 퓨샷 시연을 필요로 한다   
	- 학습된 모델에 질문을 이용해서 튜닝을 할 수 있음.
- (C) Instruction tuning 지시 튜닝은 제로샷 교차 과제 일반화를 가능하게 한다
	- 학습되지 않은 질문에 대한 능력을 키우는 것이 목적




### Instruction Tuning vs. Pretraining
- 지시 튜닝과 사전훈련은 모두 멀티태스크 학습 패러다임이다    
- 감독 신호    
	- 사전훈련: 자기지도학습(인간 주석이 없는 원시 데이터)    
	- 지시 튜닝: 지도학습(인간이 주석한 응답)    
- 과제 형식    
	- 사전훈련: 과제가 암시적이다(다음 토큰 예측)    
	- 지시 튜닝: 과제가 명시적이다(자연어 지시문으로 정의됨)    
- 목표    
	- 사전훈련: LM에게 광범위한 언어 패턴과 일반 지식을 가르친다    
	- 지시 튜닝: LM에게 특정 지시를 따르고 다양한 과제를 수행하도록 가르친다


### FLAN: Collection of Instruction Tuning Datasets
- 광범위한 이해 및 생성 과제를 포함하는 62개 데이터셋(12개 과제 클러스터)

- Instruction tuning을 할수 있게 해주는 공개데이터 중에 제일 큰 DB
	- 굉장히 다양한 종류의 데이터 셋을을 공개 해 놓음


### Generalization Improves with More Clusters
- instruction 튜닝에서 세 개의 클러스터를 의도적으로 제외했다: 상식, NLI, 폐쇄형 QA    
	- 이 3가지 클러스터에 대한 평가를 y축
- 지시 튜닝에 더 많은 클러스터와 과제를 사용할수록 보지 못한 클러스터에 대한 일반화가 더 좋아진다

- 제외한 3가지 클러스터에 대해서는 모델이 학습을 한적 없는데 다른 클러스터를 학습할 수록 학습안된 클러스터에 대해 성능이 올라감을 확인
	- 일반화 성능의 향상.
- 기 학습된 지식의 상실은 튜닝할때 너무 상이한 데이터로 하게 되면 이런 일이 발생. 
	- instruction 튜닝시에는 너무 벗어나지 않는 내용으로 하는게 좋음
	- 만약에 지식의 상실이 발생하면 사전 학습에 사용했던 오브젝트도 같이 instruction 튜닝에 사용하는 것도 방법이 될 수 있음.


### Instruction Tuning with Different Model Sizes
- 지시 튜닝은 작은 모델($<8\mathrm{B}$)의 일반화를 저해할 수 있다    
- 지시 튜닝은 큰 모델의 일반화를 상당히 향상시킨다

- 모델크기가 작으면 과적합이나 일반적인 언어 모델 능력 상실 될 때도 있음.
- instruction 튜닝은 70B 이상의 모델 사이즈에서 사용하면 좋다는 것이 그래프로 보임.


### Chat-style Instruction Tuning
- 지시 튜닝은 다중 턴 대화를 위한 챗봇을 만드는 데에도 사용될 수 있다    
- 지시문은 반드시 하나의 NLP 과제에 엄격히 대응하지 않을 수 있으며, 인간다운 대화를 모방한다    
- 다중 턴 지시 튜닝 훈련 데이터 예시: 
{"role": "user", "content": "오늘 날씨는 어때?"}, 
{"role": "assistant", "content": "화창하고 최고 기온은 75도입니다."}, 
{"role": "user", "content": "좋아! 내일은 어때?"}, 
{"role": "assistant", "content": "내일은 부분적으로 흐리고 최고 기온은 72도입니다."}`
