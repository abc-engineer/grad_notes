### Overview: Language Modeling
- 랭귀지 모델링은 어떤 단어를 얻을 확률
- 문장이 주어졌을때 다음 문장이 어떤 문작이 올지 확률적으로 이해
- 목표: 단어들의 시퀀스에 확률을 할당하는 것
- 텍스트 이해 측면:  
    p(“The cat is on the mat”) ≫ p(“Truck the earth on”)  
    → 자연스럽고 의미 있는 문장은 더 높은 확률을 갖는다.
-  텍스트 생성 측면:  
    p(w | “The cat is on the”) → “mat”  
    → “The cat is on the”라는 문장이 주어졌을 때,  
    다음 단어 w의 확률을 계산하면 가장 가능성 높은 단어는 “mat”이다.
- 다양한 어플리케이션이 있음
- 수식 증명 LLM도 있음
- 모든 자연어처리(NLP) 작업은 텍스트를 입력으로 받아 텍스트를 출력하는 문제로 변환할 수 있다.
	- LM은 기본적으로 감정분석, 기계 번역, Q&A등을 수행함.
	- in 다음 지역 나오는 등등
- 이 모든 작업들은 언어 모델링 문제로 정식화(formulated)할 수 있다.

### Probability Decomposition
- 확률을 여러 개의 더 작은 확률로 나누어 표현하는 방법
- 복잡한 확률을 조건부 확률이나 사건의 분할등을 이용해 계산하기 쉽게 분해하는 것
	- 곱셈법칙 (Chain rule / Multiplication rule)
	- 전체 확률의 법칙 (Law of Total Probability)
- Autoregressive assumption
	- 각 단어를 관측할 확률은 이전 단어에만 기반하는 추론.
	- 각 단어의 확률은 오직 이전에 나온 토큰들에만 의존한다.
	- 미래 토큰은 영향을 주지 않음
	- 전통적 모델에서 많이 사용
	- 트랜스퍼 모델에서는 다른 가정이 사용 됨.
	- 수식
		$x=[x_1,x_2,\dots, x_n$] 일때, $p(x)$
		$p(x)=p(x_1)p(x_2\mid x_1)p(x_3 \mid x_2,x_1) \dots p(x_n \mid x_1, x_2, \dots, x_n-1)$
		$=\displaystyle\prod_{i=1}^{n} p(x_i \mid x_1, x_2, \dots, x_{n-1})$
- 다른 확률 분해 모델도 가능하지만 전통적인 언어 모델로 간주하진 않음.
- 이번강의에서는 embedding, BERT등을 살 펴 볼 것.
- 문장을 관측할 확률을 작은 확률로 decomposition 함
- 확률 분포(probability distributions)는 2가지의 필요 요건이 있음. 
	- non -negative: 단어가 등장할 확률이 음수일 수는 없음.
	- 모든 확률의 합계는 1: 확률 분포의 기본 성질
- 이전 확률이 주어졌을때, 다음 w가 올 확률을 계산 
- 언어 모델의 목표는 $p(x_i \mid x_1, x_2, \dots, x_{n-1})$ 분포(distribution)를 학습하는것 
- 좋은 랭귀지 모델이 있다고 가정
	- 그러면 다음 토큰을 하나씩 생성 할 수 있다.
	- 과거 단어를 기반으로 다음 단어를 잘 예측 할 수 있다는 뜻
	- 예측이 되면 생성을 잘 할 수 있음
	- 샘플링이나 greedily(가장 높은 확률을 고르는 것)를 할 수 있음
		- Sampling: 언어모델이 다음 단어를 선택할 때,  확률 분포에서 무작위로 선택하는 방법
		- greedily: 매 단계에서 가장 확률이 높은 단어만 선택하는 방법
			- $x_i=arg\ max_wP(w∣x_1,...,x_i−1)$
	- 다음 단어를 계속 예측하는 것이 생성모델의 기본
- 생성을 종료하는 방법
	- 특정 기호가 나타나면 생성을 종료함.
	- EOS: end-of-sequence가 생성되면 종료하는 심볼을 사용함.

### N-gram Language Models
- 이전 토큰을 트랙킹 하는게 어려움.
- 이전의 모든 토큰을 추적하는 대신, 한 단어의 확률이 오직 이전 N−1개의 단어에만 의존
- 어떤 문장을 관측할 확률을 decompose (Autoregressive assumption 이용)
- 예측을 하려면 앞에 나온 단어를 다 계산 해야 했음.
- model long contexts at all 불필요
- n-1개의 단어만 계산을 해서 확률을 단순화 하자
- N-gram의 뜻: 하나의 확률을 계산 할때 n개 까지만 보겠다는 뜻
	- N을 몇으로 하는지에 따라 모델의 성능과 성향 달라짐.
- Unigram: N=1
- Bigram: N=2
- Trigram: N=3
- 과거 단어는 N-1개만 계산
- 확률을 decompose후 어떻게 값을 얻을까?
	- 기존에는 확률 빈도를 이용해서 estimated
	- 과거의 context를 세고 지금 단어와 과거의 context를 세서 계산(maximum likelihood estimation)
	- Unigram: 각 단어나 나온 개수를 셈
	- Bigram: the cat이란 문장이 있을때 
		- \#the cat / \#cat

### Practice: Learning Unigrams
- unigram은 각 단어를 다 세지 않고 지금 단어만 셈
	- 단어간의 관계를 고려하지 못한다는 치명적인 단점이 있음
	- the the the the라는 문장 보다 I have a cat 이 더 자주 나오는 문장으로 계산 됨
	- 그래서 바이그램 등장
- 단순히 개수만 세면 되지만 효과적인 방법 임.
- 수식
- $p(x_i)=\displaystyle\frac{\#(x_i)}{\#(all\ counts\ in\ the\ corpus)}$

### Practice: Learning Bigrams

- 수식
	- $p(x_i \mid x_{x-1})=\displaystyle\frac{\#(x_{i-1}, x_i)}{\#(x_{i-1})}$

- 제목 오타임 바이그램 예시
- 예시 코퍼스에 a cat 말고 the cat이 나왔는데 이렇게 학습데이터에 a cat이 나오지 않으면 문법상 맞는 문장이지만 확률이 0이되는 문제가 생김
	- Sparsity: 희소성 문제라고 함.
	- 학습 데이터에 등장하지 않았다는 이유로 유효한 바이그램의 확률이 0이 되는 문제
- 엔트리 (테이블)를 보면 0으로 많이 채워 져 있음.


### Practice: Learning Trigrams
- 확률 계산은 쉬움
-  - 수식
	 - $p(x_{x-2} \mid x_{x-1}, x_i)=\displaystyle\frac{\#(x_{i-1}, x_i)}{\#(x_{x-2}, x_{i-1})}$
- 이전 단어 2개를 가지고 지금 단어가 나올 확률 게산.
- the cat 다음에는 is나 eos 2중 1개가 나온다는 것을 알 수 있음
	- 하지만 실제 언어에서는 the cat 다음에는 무수히 많은 단어가 올 수 있음
- 이번에도 Sparsity문제가 발생
- N becomes larger의 특징
	- N 이 커질 수록 이론적으로는 더 좋은 모델링을 할 수 있지만
	- 희소성 문제가 더 발생함.
- 보카가 늘어날 수로 희소성 문제는 지수 함수로 증가
- 가능한 N-그램(파라미터)의 수는 N이 증가할수록 기하급수적으로 증가
- N이 많을 수록 모델은 더 specific해짐.
- N-gram in the training data are lower


### Smoothing in N-gram Language Models
#### Overcoming Sparsity in N-gram Language Models
- N-gram의 한계점:  zero-probability
	- 학습 데이터에 등장하지 않은 N-그램은 항상 확률이 0이 된다. 
	- 어떤 항이라도 확률이 0이면 전체 시퀀스의 확률도 0이 된다.
- N그램은 간단하게 언어 모델을 만들 수 있지만, 관측되지 않은 데이터는 0으로 계산되는 치명적이 단점이 있음.
- 이를 보완하는 방법이 스무딩.
- n-gram이 non-zero가 되게
	- 관측되지 않았다고 0으로 하지는 말자.

#### Smoothing
-  Intuition(직관): 학습 데이터에서의 등장 횟수와 관계없이 모든 N-그램이 0이 아닌 확률을 갖도록 보장한다.    
- 스무딩 기법:  
	- 애드원 스무딩(라플라스 스무딩)  
    - 애드-k 스무딩  
    - 언어 모델 보간  
    - 백오프
#### Add-one Smoothing (Laplace Smoothing)
- 모든 엔트리에 그냥 1을 더하는 방법
- 대소 관계는 유지하면서 0이 되는 엔트리를 없앨 수 있음음
- 각가의 분자에 1이 더해지고 분모에는 voca size를 더함.
	- xi가 v번 더해 지니까 분모에 더해짐
- 오버 스무딩 문제가 생김
	- 안나올 것 같은은 n-gram도 값을 갖게 됨.

#### Add-k Smoothing
- Laplace Smoothing은 모두 1을 더하다보니 너무 과하니까 1보다 작은 특정 수를 더하자.
- 소수값 k(k<1)를  더함. (일종의 하이퍼 파라메터임)
	- 밸리데이션 세트를 이용해서 k값을 정함.


#### Recap: Validation Set
- 훈련 데이터: 트레이닝에만 사용
	- labeled data에서 패턴을 학습
- 평가 데이터: 모델 평가에 사용
	- unlabeled data의 라벨을 에측
- 밸리데이션 데이터: 개발 중에 얼마나 잘 개발 되고 있는지 사용하는 데이터
	- 모델이 얼마나 잘 작동하고 있는지 주기적으로 평가
	- 밸리데이션 데이터는 학습에 사용되지 않음
	- 검증 세트는 테스트 데이터를 직접 사용하지 않고 모델 성능을 평가하기 위해 학습 데이터에서 따로 분리해 둔 일부 데이터이다.
	- 이는 테스트 세트를 대신하는 역할을 하며(즉, 보지 못한 데이터처럼 모의로 사용됨), 모델의 일반화 성능을 확인하는 데 사용된다.
- 밸리데이션에서 가장 낮은 loss를 갖을 때 학습을 멈춤
- 하이퍼파라미터도 밸리데이션 세트를 갖고 정해야 함.

#### Smoothing via Language Model Interpolation
- Interpolation을 이용한 스무딩 모델
- 서로 다른 N-그램의 장점을 결합
- n이 작으면 context를 많이 못보지만 희소성 오류는 적음
- n이 크면 반대로 많은 context를 포착(capture)하지만 희소성 오류 많음
- n이 클때, 작을때 장단점이 명확하니 섞어서 사용하자.
- n이 1~ large n일때 까지 계산 해서 앙상블 모델을 만듦.
- $\lambda_n$도 하이퍼 파라미터이고 밸리데이션 세트를 이용해서 정함.

#### Smoothing via Backoff
- 백트레킹 리서치와 비슷함.
- 제일 큰 n-gram에서 먼저 보고 관측이 안되면 n-1로 낮춰서 context를 봄
- 가장 높은 차수의 N-그램부터 시작한다.    
- 해당 N-그램이 존재하지 않으면(빈도가 0이면), 한 단계 낮은 (N-1)-그램을 사용한다.    
- 확률이 0이 아닌 N-그램이 나올 때까지 더 낮은 차수의 N-그램으로 계속 내려간다.
- 수식으로 쓰면 if ㅁㅁㅁ>0이면 쓰고, <0이면 n-1
- backoff를 적용해서 n=1일때 까지 봤는데 확률이 여전히 0일수 있을까?
	- 0일수 있음
	- corpus에 등장하지 않았다면 여전히 0이라는 확률이 나올 것. -> Out-of-vocabulary Words

### Out-of-vocabulary Words: OOV
- 한번도 등장하지 않은 데이터나 단어를 말함.
- 유니그램 모델에서는 학습 데이터에 등장하지 않은 단어에 대해 확률이 0이 된다.    
- 간단한 해결 방법: 미지/미관측 단어를 위한 특수 토큰 \[UNK]를 사용한다.  
    - 테스트 시에는 모르는 단어를 \[UNK]로 변환하여 \[UNK]의 확률을 사용한다.    
- \[UNK]의 확률은 어떻게 추정하는가?
    - 학습 과정에서 희귀 단어들을 \[UNK]로 치환하고, 이를 일반 단어처럼 간주하여 확률을 추정한다.    
- 희귀 단어를 어떻게 결정하는가?  
    - 학습 데이터에서의 빈도를 기준으로 임계값을 설정한다.  
    - 예: 어휘 크기를 1만 개로 제한하고, 상위 1만 개를 제외한 나머지 단어들을 학습 시 \[UNK]로 변환한다.
- 예를 들어 CNN이 등장하기 전에 만들어진 모델은 CNN이란 단어의 확률=0임
- 신조어는 무조건 확률 0
- 이런 단어는 어떻게 핸드링 할지가 요즘도 관건임.
	- 그냥 스페셜한 언노운 토큰을 두고 신조어는 언노운 토큰으로 값을 리턴하는 방법을 많이 사용
	- 언노운 토큰의 확률은 어떻게 처리?
		- 특정 개수 이하로 나온 단어를 추림(Threshold 이하) -> 일일히 세지 않고 언노운 토큰으로 치환
		- 이 언노운 토큰의 개수를 세서 값을 할당함.
		- 확률이 커질 수 있어서 Threshold를 보수적으로 잡는게 좋음.
		- 언노운 토큰으로 치환된 단어는 생성에서 배제 되도록 처리.

### Evaluation of Language Models
- 언어 모델이 얼마나 좋은지 어떻게 평가할까?
	-  완벽한 언어 모델은 말뭉치의 모든 단어를 정확하게 예측할 수 있어야 한다.  
    - 우리는 언어 모델이 다음 단어에 높은 확률을 부여하기를 기대한다.  
    - 더 좋은 언어 모델일수록 다음 단어에 대해 “덜 놀란다”(예측을 더 잘한다).
- 아이디어:  
    - 언어 모델이 부여한 다음 단어의 확률을 그대로 평가 지표로 사용하자.
- 완벽한 모델이라면 코퍼스에서 모든 다음 단어를 예측할 수 있어야 함.
- less surprised해야함.
- 언어모델을 결국 다음 단어를 예측하는 것.
	- 다음단어를 얼마나 잘 예측했는지를 평가 지표로 사용.

### Training/Validation/Test Corpus
- Corpus: 말뭉치
- 평가를 위해 ML 모델과 동일하게 training/validation/test set로 나눠서 훈련 및 평가
- 학습 말뭉치/데이터셋: 모델을 학습시키는 데 사용하는 텍스트 데이터    
- 학습 말뭉치에서 언어 모델의 확률을 평가하는 것이 타당한가?    
- 학습 데이터로 평가하면 다음 단어 예측 확률이 과도하게 높게 나와 오해를 일으킬 수 있다 → 학습-테스트 데이터 누수 문제    
- 테스트 말뭉치/데이터셋: 학습 데이터와 겹치지 않도록 따로 분리해 둔 데이터    
- 모델 성능 평가는 반드시 테스트 말뭉치를 사용해야 하며, 이를 통해 보지 못한 데이터에 대한 일반화 능력을 측정한다    
- 테스트 데이터는 하이퍼파라미터 튜닝이나 설계 선택을 위해 반복적으로 사용하면 안 된다 → 간접적으로 테스트 데이터의 특성을 학습하게 됨    
- 검증/개발 말뭉치/데이터셋(선택):  
    테스트 평가 전에 하이퍼파라미터 조정과 설계 결정을 위해 사용하는 데이터셋
- 데이터가 고정되어 있다면 학습/검증/테스트 세트를 어떻게 나눌 것인가?    
- 학습 세트는 가능한 한 크게 유지하는 것이 좋다.    
- 그러나 검증 세트와 테스트 세트도 신뢰할 수 있는 평가를 위해 충분한 크기를 가져야 한다.    
- 테스트 세트는 실제로 언어 모델을 적용하려는 데이터나 작업을 잘 반영해야 한다.

### Perplexity
- 얼마나 예측가능한 범주에서 생성을 하는지 평가
	- Perplexity(PPL)으로 평가
	- 언어 모델의 성능을 평가하는 지표
- Perplexity(약어: PPL)는 언어 모델을 평가하는 내재적 평가 지표이다
- test 문장(test set에 있는 문장)을 가지고 연산을 수행.
	- 컨텍스트가 주어졌을때 다음 단어를 예측
	- test text를 잘 설명하면 분모가 커지니까 PPL은 작아짐.
	- 예측가능한 범주에서 텍스트를 에측
	- n-gram에 따라 분모는 당연히 달라짐.
	- GPT도 아직 PPL을 이용해서 모델을 평가 하고 있음.
- raw probability scale값으로 PPL을 계산하면 수치적 불안정성이 발생할 수 있다.
- 따라서 실제로는 로그 스케일에서 PPL을 계산하는 것이 일반적이다.
- 이것보다 디테일이 더 있긴 함.
- n번의 곱이 생기는데 n이 작아지면 수학적 안정성이 너무 떨어짐(너무 작은 결과)
- -> log와 sum을 사용해서 보완
- PPL은 중요한 랭귀지 모델의 벤치마크 지표
- GPT2 시절 metric(지표)
- LSTM모델도 있음

### Intrinsic vs. Extrinsic Evaluation (내재적 vs. 외재적)
- 내재적 지표(intrinsic metrics, 예: perplexity)는 특정 응용과는 관계없이, 언어 모델링 자체의 품질을 직접 측정한다.
	- 텍스트를 잘 이해 하는지 평가
- 외재적 지표(extrinsic metrics, 예: accuracy)는 특정 작업이나 응용(예: 분류, 번역)에서 언어 모델의 성능을 측정한다.
	- 구체적 어플리케이션에 적용해서 평가
	- 번역, 감정 평가등에 적용해서 정확도가 얼마나 나오는지 평가.
- 최근 언어모델은 PPL만 평가하지 않고 수학문제도 풀어보고 다양하게 평가함.
- intrinsic값과 extrinsic 값이 비례하진 않고 모델에 다라 달라짐
- extrinsic metrics 결과 예시 (라마3 출시시절 데이터.)

### Summary: Language Modeling
- 전통적인 기법으로 N-gram 대표적으로 소개
- 지금은 LLM시대라 전통적 기법을 깊게 할 필요는 없음.
- n이 커졌을때, 작아졌을 때 어떤 특징이 있는지 잘 알아 둘 필요 있음.
- 언어모델의 평가는 중요한 챕터임.
	- Training/validation/test split
	- Perplexity
	- intrinsic extrinsic등의 개념은 꼭 알고 넘어 갈 것.
- 언어 모델링은 자연어처리의 핵심 문제이다.    
- 모든 자연어처리 작업은 언어 모델링 문제로 정식화할 수 있다.    
- (Autoregressive) 언어 모델은 텍스트 생성에 활용될 수 있다.    
- 언어 모델의 확률 분포는 학습 말뭉치를 통해 추정(학습)된다.

### Summary: N-gram Language Models
- N-그램 언어 모델은 (일반적인) 언어 모델 가정을 단순화하여, 한 단어의 확률이 오직 이전 N−1개의 단어에만 의존한다고 본다.    
- 낮은 차수의 N-그램(작은 N)은 문맥 정보와 단어 간 상관관계를 적게 반영한다.    
- 높은 차수의 N-그램(큰 N)은 희소성 문제가 더 심해지고, 파라미터 공간이 매우 커진다.    
- N-그램 언어 모델의 희소성 문제를 해결하기 위해 스무딩 기법을 사용할 수 있다.  
    - 애드원 스무딩  
    - 애드-k 스무딩  
    - 언어 모델 보간  
    - 백오프

### Summary: Language Model Evaluation
- 언어 모델을 학습하고 평가하기 전에 학습/검증/테스트 데이터 분할이 필요하다.    
- 퍼플렉서티는 언어 모델이 다음 단어에 대해 얼마나 “혼란스러워하는지”를 측정하는 지표이다.    
- 테스트 세트에서 퍼플렉서티가 낮을수록 더 좋은 언어 모델이다.    
- 퍼플렉서티는 언어 모델링에서 널리 사용되는 내부 평가 지표이다.    
- 퍼플렉서티는 실제로 로그 스케일에서 계산된다.   
- 내부 평가와 외부 평가는 모두 중요하다.
