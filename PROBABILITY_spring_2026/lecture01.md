### Probability vs. Statistics
- 확률 수업이지만 통계는 항상 따라옴.
- 둘다 랜덤 프로세스지만 둘은 매우 다름
- 확률
	- 논리적으로 완벽
	- 부족한게 없음
	- 확률 계산 법칙이 몇가지 있음
	- 문제의 답은 1개
	- 수학에 더 가까움
- 통계
	- 조금 더 어지럽? 더럽고 예술에 가까움
	- 확률 기반 추론을 통계라고 함.
	- 하나의 답이 있는 것이 아님. 다양한 답이 있을 수 있음
	- 통계학자는 서로 다른 의견을 가질수 있음


### 확률 예시
- 앞 뒤 확률이 같은 fair coin이 있을 때 100번 던지면 앞면이 60번 이상 나올 확률은?
	- 2.8% 라고 계산 할 수 있음
- 무작위 과정은 정해져 있음 (확률 0.5)
- outcome은 60개 이상의 앞면이 나올 것.
- 이 과정이 확률이 하는 일.
- 나중에 이항 분포로 계산



### 통계학 예시
- 데이터를 통해 모르는 것을 추론하는 학문
- 통계적 모델링은 매우 다양
- 결론을 얻기 위해 확률을 계산이 들어감.
- 공학에 더 가깝다고 봄
- 랜덤 프로세스를 더 클리어하게 추론하는게 통계학

### Frequentist vs. Bayesian Interpretations
통계의 유명한 두가지 이론
요즘 통계학자들은 둘다 잘 다룸

#### Frequentist
- 확률은 실험의 빈도 수를 재는 것
- 생물학, 약학, 사회학에 많이 사용
	- a=0.01 어쩌구 저쩌구

#### Bayesian
- 확률이 빈도수라기 보다는 얼마나 확신을 갖고 있느냐로 해석
- 동전이 앞면이 나올 확률이 50%다 이런 값을 대입하지 않고
- 5:5 동전이 아니고 40~60%에 들어갈 확률이 70%의 신뢰를 갖고 있다라고 해석
- 컴퓨터, 빅데이터에서 많이 사용

### Sets and Notations
집합 S는 원소들의 모음
- 원소(Element)
	- x는 집합 S의 원소 
	- x ∈ S
- 부분집합(Subset)
	- A의 모든 원소가 S에 속해 있을 때
	- A ⊂ S
- 여집합(Complement)
	- A가 S 안에 들어있을때 A가 속해 있지 않는 집합
	- $A^c$ or S − A
- Union: 합집합
- intersection:교집합
- Empty set:공집합
- Disjoint: 서로소, 교집합이 공집합
- Difference: 차집합
	- 교환법칙 성립 안함.


### 드모르간의 법칙(DeMorgan’s laws)

- 합집합의 여집합은 여집합의 교집합  
	$(A ∪ B)^c = A^c ∩ B^c$
- 교집합의 여집합은 여집합의 합집합  
	$(A ∩ B)^c = A^c ∪ B^c$




### Products of sets
set의 곱(집합의 곱)
- s와 t가 각각 집합의 원소일때 둘의 곱은 페어로 쓸 수 있음
- {1,2,3}x{1,2,3,4} = 12개의 쌍
- 연속되는 값도 곱이 가능함. [1, 4] × [1, 3]



### Counting
- set S가 유한하면 ∣S∣ or # S 라고 씀
- Inclusion-exclusion principle
	- 합집합에서는 교집합의 개수를 빼줘야 중복 없이 카운팅
	- ∣A ∪ B∣ = ∣A∣ + ∣B∣ − ∣A ∩ B∣



#### 곱셈 법칙
- 액션 1은 n가지, 액션 2는 m가지면 둘의 곱이 경우의 개수
- 액션 1,2 가 인디펜던트일 필요가 없음 (독립일 필요가 없음)
	- 5명의 선수가 올림픽에서 금은동을 딸 경우의 수는 몇개인가?
	- 5x4x3=60가지 (금메달을 1명주면 은메달을 딸 수 있는 사람은 당연히 4명 동은 3명)


### Permutations
- 집합의 원소가 가질수 있는 순열의 가지수= 원소의 순서 개수
- 나열해서 구할 수 있지만 당연히 계산으로 구할 수 있음
- k개 원소는 k!의 순서를 갖음
	- {a,b,c}는 6개의 permutation
	- 3x2x1=6 (3!)
- n개의 원소에서 k개 원소의 permutation을 구할 수 있음
	- 개수가 늘어남.
	- 4개의 원소 중 3개의 원소를 고르는 경우 4x3x2 = 24 가지


### Combinations
- permutation과 다르게 순서가 중요하지 않음
- 집합이라고 보면 됨 permutation은 리스트 느낌
- Permutation에서 순서만 다른 것들을 묶어주면(나눠주면) 됨


### Formulas

- Permutations: ${}_nP_k = \frac{n!}{(n-k)!} = n(n-1)\cdots(n-k+1)$
- Combinations: ${}_nC_k = \binom{n}{k} = \frac{n!}{k!(n-k)!} = \frac{{}_nP_k}{k!}$
