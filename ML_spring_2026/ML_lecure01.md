### ML
- Rule-based expert systems의 한계를 극복하기 위해 ML 연구
- expert systems의 단점이 ML의 장점임.
- 유연성이 큼

- 종류가 매우 다양함.
- 이번 학기에 하나씩 살펴 볼 예정
- 교수님은 사람의 인지과정에 관심이 매우 있음

### Three main element of ML
- 데이터: 데이터가 기본, 지식
	- 데이터는 지식을 추출할 수 있는 예시나 관찰값을 제공
- 알고리즘: 소프트웨어라고 보면 됨
	- 알고리즘은 데이터를 처리하고, 유용한 패턴을 어떻게 포착할지 알아내는 절차이다.
	- 어떤 사람은 빠르게 누구는 느리게
	- 학습 방식이라고 볼 수 있음
	- 알고리즘에 따라 ML이 달라짐
	- 어떻게 훈련을 시킬지 목적에 따라 모델의 퍼포먼스가 매우 달라짐.
- 모델: 아키텍쳐
	- 모델은 알고리즘에 의해 학습된 함수로, 입력을 출력에 대응시키며 예측에 사용될 수 있다.
	- 인풋을 어떻게 아웃풋으로 내뱉는지
	- 일종의 function
	- 사람과 비교 했을 때 사람이 물통을 보면 눈으로 신호를 받아서 뇌로 전달하고 이게 물통이라고 인지 하게 되는데 물통이라고 인지하는 과정을 모델이라고 보면 됨.
	- 실제 뇌에서도 인지 부분은 복잡한 영역
	- 모델은 인풋과 아웃풋을 연결하는 함수
		- 반면 알고리즘은 펑션이 같아도 목적이 다르게 할 수 있음.
		- 물통을 물통이라 명명할 수도 있고, 무기로 쓴다라고 판단 할 수도 있음.


### Three main approaches to machine learning.(ML 카테고리)
- 카테고리는 매우 다양하지만 이 3가지가 기본
	- Supervised learning
	- Unsupervised learning
	- Reinforcement learning
		- 이번 수업에는 비 대상.

### Supervised learning
- 지도학습은 크게 2개
	- Classification, Regression이 대표적
- 감독이 있음
- 데이터가 주어지고 정답값이 주어짐
- 정답= label
- input, output이 주어진 ML
- 목적은  labeled data를 학습해서 정확한 예측을 하는 것.



#### Classification
- 분류(classification)는 과거의 관측값을 바탕으로 새로운 데이터 포인트의 범주형(categorical) 클래스 라벨을 예측하는 데 사용
- 분류하는 작업
- 카테고리가 2개면 바이너리 classification
	- 스팸, 비스팸 등
- 객체인식 처럼 카테고리가 여러가지가 있으면 multi-class calssification
- 레이블은 주어져야 함.

#### Regression
- 회귀(regression)는 설명변수와 연속형 결과값 사이의 관계를 모델링하여 숫자 값을 예측하는 방법
- 리니어 모델 대표 - 다음주 실습 예정
- 회기(리그레션)
- 아웃풋 자체가 연속적인 경우가 리그레션
	- 분류와 비슷하지만 아웃풋이 연속임
	- 값이 카테고리면 분류
- 예시
	- 집값 예측: 집값이 연속 값이므로 카테고리 분류는 안어울림
	- 아이 성장
	- 의료진단 등등



### Unsupervised learning
- 크게 보면 라벨이 없음
- 데이터만 보고 데이터의 숨겨진 패턴이나 구조가 생성되었는지 찾아 학습하는 것.
- 학기말에 많이 다루게 될 예정
- 정리함에 라벨이 없으면 어떻게 정리할지 아이가 나름의 기준을 만드는 것과 유사함.
- 기준에 따라 비지도 학습의 특성과 성능이 많이 달라짐.


#### Clustring
- 아주 많이 사용 되는 task 중 하나
- 피쳐를 보고 유사도에 근거해서 비슷한 것 끼리 군집을 나누는 것.
- 클러스터링(clustering)은 각 데이터가 어떤 그룹에 속하는지에 대한 사전 정보 없이, 정보를 의미 있는 그룹으로 조직하는 방법
- 중요한것은 라벨이 주어지지 않음.
- 예시
	- 프랜드쉽 그룹핑
	- 레스토랑 메뉴 클러스터링
	- Customer segmentation


#### Dimensionality reduction
- 차원 축소법
- 종종 사용됨.
- 여러가지 모델을 다룰 예정
- 리얼 월드, 실 생활은 고차원 데이터 속에서 살고 있음
	- 수업 구성원만 봐도 색깔, 형태 등등 다양함
	- 이것들을 효율적으로 기억하려면 중요 특징으로 나눠 기억할 수 있음
		- 남여, 반팔 긴팔등
- 데이터도 마찮가지로 모델을 축소하면 데이터 다루기도 편하고 사용하기 편하고 성능도 좋음




### Reinforvement learning
- 종종 지도/비지도 학습과 다른 카테고리로 보기도 함
- 지도학습같은 레이블은 없음
- 하지만 레이블이 아예없진 않음
- ML이 결과를 토대로 스스로 보상을 통해 학습을 해나가는 모델
- 에이전트(agent)는 환경과 상호작용하면서 시행착오(trial and error)를 통해 학습하고, 이때 보상 신호(reward signals) 형태의 피드백을 받는다. 일반적으로 에이전트의 목표는 장기적인 보상을 최대화하는 전략을 찾아내는 것
- Q: 이 영상이 강화 학습인 이유??
- A: 솔직히 잘 모르겠음
	- 사람이면 반찬 밥등을 나눠 자연스럽게 넣겠지만 기계는 판단 못함.
	- 액션을 수십번 반복하여 패턴이나 규칙을 학습했을때 강화 학습이라고 판단.

### Challenge of Generalization
- 머신러닝의 이슈 중 가장 큰게 일반화 문제임
	- 모델을 훈련하고 테스트 세트에 적용시 새로운 세트에 실패하는 경우가 종종 있음.
	- 테스트와 트레이닝 셋과 일치 않을 수도, 모델이 잘못 되었을 수도 있음.
- 머신러닝의 핵심 과제는, 모델이 학습에 사용된 입력 데이터에서만 잘 작동하는 것이 아니라 이전에 보지 못한 새로운 입력 데이터에서도 잘 작동해야 한다는 점이다.  
- 이렇게 이전에 관찰되지 않은 입력에 대해서도 잘 성능을 내는 능력을 일반화(generalization)라고 한다.
- 새로운 세트(or test set)에 잘 적용 되면 일반화 성능이 높다고 판단
- 고양이vs강아지 구분은 쉬워 보이지만 학습을 위해 컴퓨터에게 설명하기 어려움
	- 모델에 따라 분류 잘 되기도 안되기도 함.


### Divide dataset
- 실제 데이터는 테스트 세트가 나워져 있지 않음
	- 트레이닝 세트와 테스트 세트를 나눠야 함.
- 실제로는 중간에 밸리데이션 세트를 만드는 방법을 많이 사용함.
	- Training set: 모델을 학습시키고 데이터 안의 패턴을 포착하는 데 사용
	- Validation set: 개발 과정에서 모델 선택과 매개변수 조정을 위한 기준으로 사용
	- Test set: 훈련 세트와 검증 세트와는 분리해 두며, 모델이 실제 환경의 데이터에서 어떻게 동작할지 예측하는데 사용

 
#### Cross-validation 
 - single train-validation 보다 크로스 밸리데이션을 하는것이 좋음
	- 데이터셋을 여러 개의 부분으로 체계적으로 나누어, 각 데이터 포인트가 서로 다른 단계에서 트레이닝과 밸리데이션에 모두 사용할 수 있음
-  데이터셋을 크기가 같은 k개의 부분집합, 즉 폴드(fold)로 나눈다. 
- 모델은 총 k번 학습되며, 매번 k−1개의 폴드를 훈련에 사용하고 남은 1개의 폴드를 검증에 사용한다. 최종 성능은 모든 폴드에서 얻은 결과를 평균하여 구한다.
- 이렇게 하면 총 k 개의 밸리데이션 세트를 만들수 있음
- 이게 k-fold-cross-validation

#### Leave-one-out cross-validation
- 다양한 클로스 밸리데이션이 존재하는데 leave-one-out은 그 중 하나.
- k-폴드 교차검증의 극단적인 경우로, 여기서 k는 데이터셋에 포함된 데이터 포인트의 개수와 같다.  
- 각 반복에서는 하나를 제외한 모든 데이터 포인트를 훈련에 사용하고, 남은 하나의 데이터 포인트를 검증에 사용한다.
- 예를 들어 데이터가 100개면:
	- **5-fold**: 20개씩 나눠서 5번 검증
    - **LOOCV**: 1개씩 검증해서 100번 검증
- 단점으로 fold가 매우 다양해짐
	- 그래서 많이 사용하진 않음
	- 교수님은 4~5fold 선호




#### Stratified K-fold cross -validation
- 학기 중반에 살짝 다룰 예정
- 각 폴드에서 클래스의 분포가 유지되도록하는 k-fold 교차검증의 한 종류
- 불균형 데이터셋(imbalanced datasets)에서 유용하며, 각 폴드가 원래 데이터셋과 동일한 클래스 비율을 유지하도록 함.
	- 데이터가 불균형한 경우가 있음
	- 10개 20개이렇게 데이터 종류가 다르면 동일하게 나눌수 없음
	- 각각 폴드별로 동일한 비율을 유지하여 fold를 나누는 방법.


###  Generalization error: underfitting / overfitting
매우 중요
- 일반화 성능을 볼때 중요한 2가지 지표(문제)가 있음
- underfitting
	- 훈련데이터도 훈련이 안된경우
	- 트레이닝 데이터 보다 테스트 결과가 낮은 경우가 언더 피딩
	- 데이터는 복잡한데 모델은 너무 단순한 모델을 사용
		- 위예시처럼 복잡한 데이터에 1차 함수 모델 사용함.
- overfitting
	- 반대로 트레이닝은 잘되 었는데 테스트 데이터 결과에는 값이 안 좋음
	- 피팅이 너무 되어 over
	- 실제 데이터에 비해 너무 복잡한 모델을 사용.
		- 훈련데이터는 값을 잘 예측
		- 테스트 데이터에서는 잘 예측 못함.
		- 훈련데이터의 노이즈 까지 학습하는 경우가 있음음
- 모델의 목표는 true function을 잘 예측하는 것.


### Model control
- 오버/언더 피팅 방지법
- 모델의 capacity(용량) 를 올리거나 내림
- 신경망의 레이어를 줄이거나 파라메터, 피쳐 를 줄이면 복잡도 하락, 반대는 상승
- capacity가 낮은 모델은 훈련 데이터의 패턴을 충분히 포착하는 데 어려움을 겪는 경우가 많고, 반대로 capacity가 높은 모델은 훈련 세트의 세부사항을 지나치게 암기하여, 보지 못한 데이터에는 잘 일반화되지 못하는 과적합의 위험이 있음





### 일반화 성능 보장하는 방법
- 트레이닝 에러와 밸리데이션 에러(제너레이션 에러)를 동시에 측정
- 노이즈를 학습하면 밸리데이션 에러가 올라가기전에 학습을 중단 함.





### bias-variance tradeoff
- 언더피팅, 오버피팅이라 안하고 어떤 필드에서는 바이어스, 배리언스 에러라고도 함.
- bias 에러(편향 오류)
	- 학습이 잘 안됨.
	- 학습 알고리즘의 잘못된 가정에서 비롯되는 오류
	- 모델의 가정이 지나치게 단순할 때 발생하며, 이로 인해 모델이 데이터의 실제 패턴에서 지속적으로 벗어나게 되어 underfitting
- variance 에러(분산오류)
	- 노이즈 까지 학습
	- 입력의 작은 변동에도 민감하게 반응함으로써 발생하는 오류
	- 자신이 학습한 특정 데이터에 지나치게 민감할 때 발생한다. 분산이 큰 모델은 훈련 데이터에 따라 결과가 크게 달라지며, 실제 패턴이 아니라 무작위 잡음(random noise)까지 학습하게 되어 overfitting
- variance error 내려가는 방향은 언더 피팅 올라가는 방향이면 오버 피팅 존
- 낮은 바이어스 에러, 낮은 배리어스 에러인 지점을 찾는게 목적

### Training process에 따른 분류(에폭 수)
- 얼마나 훈련을 여러번 시켰는지를 기준으로 나눌 수도 있음
- 모델이 너무 적게 학습되면, 이는 용량이 낮은 모델과 비슷하게 작동하여 데이터의 패턴을 충분히 포착하지 못하고 과소적합(underfitting)이 발생한다.  
-  반대로 모델을 지나치게 많이 학습시키면, 이는 용량이 높은 모델과 비슷해진다. 이 경우 모델은 훈련 데이터의 세부사항, 심지어 관련 없는 잡음(noise)과 이상치(outliers)까지 암기하게 되어 과적합(overfitting)이 일어날 수 있다.
- 두가지 에러가 어느정도는 내려 간 다음 훈련을 중지해야 하는데 에폭이 적으면 언더, 너무 많으면 오버피팅 됨.

### double-descent phenomenon
- 일반적으로 밸리데이션 밸류가 올라가면 훈련을 멈추는 것이 기본
- 하지만 double descent 되는 경우가 있음
- 아직 명확하게 연구 되었다고 확신할 수는 없음
- 발견 했다 수준의 논문 이었음
- GTP 같은 대형 모델에 비용을 계속 투자하고 성능을 올리니 오버 피팅이 아니라 계속 성능이 좋아짐.
- 교수님 연구분야는 아니어서 정확한 설명은 불가하고 핫했던 주제여서 소개


### relationship between the training and test data
#### in-distribution
- 테스트 데이터와 트레이닝 데이터가 어느정도는 유사해야함
- 테스트 예제가 훈련 세트와 유사한 특성과 패턴을 공유하면, 이를 인디스트리뷰션(in-distribution)이라고 하며, 이 경우 모델은 대체로 기대한 대로 성능을 낼 가능성이 높다.
#### out-of-distribution
- 테스트 데이터와 트레이닝 데이터가 너무 다르면 out-of-distribution
- 테스트 예제가 훈련 데이터와 크게 다를 경우, 예를 들어 학습 과정에는 없던 새로운 특징, 패턴, 조건이 포함되면, 이를 아웃오브디스트리뷰션(out-of-distribution)이라고 하며, 이때는 학습된 패턴이 더 이상 잘 적용되지 않기 때문에 성능이 자주 저하
- out-of-distribution 테스트 데이터가 나오면 이상한 값을 내보내는 경우가 있음
	- 예 새를 횡단보도로 추론
	- 제로샷? 이라고 볼 수 있음. 장르는 다름

### shortcut learning
- 인공지능은 지름길을 학습하는 것 같다는 논문
- 별과 달의 형태를 보고 학습안하고 위치를 보고 학습
- 우리가 생각하는 피쳐와 다르게 쉽게 학습하는 현상
- 이런 현상도 일반화를 실패 했다고 할 수 있음
	- 숫자 사진이 바뀌면 인식 못함.
	- 강아지 사진에 노이즈가 있으면 강아지 추론 실패
	- 윤곽선만 따져 있는 고양이를 코끼리로 인식(코끼리 피부 패턴으로 인식)

### 정리
- 일반화 문제가 매우 중요
- 언더, 오버피팅 문제를 잘 다뤄야 함.
-  일반화(generalization)는 머신러닝의 핵심에 있다. 일반화는 모델이 정말로 학습한 것인지, 아니면 단지 암기만 한 것인지를 결정하기 때문이다.  
-  좋은 모델은 단순히 훈련 데이터에 잘 맞는 것에 그쳐서는 안 되며, 학습한 내용을 새로운 상황에도 적용할 수 있어야 한다.  
-  모델이 점점 더 복잡해지고 실제 환경에서 사용됨에 따라, 신뢰할 수 있는 일반화를 위해서는 신중한 평가, 적절히 통제된 모델 용량, 그리고 훈련 데이터와 테스트 데이터의 분포에 대한 주의가 필요하다.

### 관련 수학 지식 소개
- 머신러닝은 데이터를 표현하고, 패턴을 학습하며, 모델의 성능을 최적화하기 위해 여러 수학 분야에 의존한다.  
- 이러한 모델이 어떻게 구축되고, 학습되며, 개선되는지를 더 잘 이해하려면 그 기초가 되는 수학을 배우는 것이 도움이 된다.

#### 선형대수 
- 많이 사용
- 기하학적으로 세상을 이해
- 인풋, 모델의 파라메터 등을 벡터 행렬로 표현
- 인풋 아웃풋의 관계 설정에 많은 변형이 발생함. 이럴때 선형대수 많이 사용
- 입력값, 매개변수, 그리고 중간 표현은 종종 벡터와 행렬로 표현되며, 모델은 고차원 공간에서 데이터를 회전, 확대·축소, 투영하는 선형 변환을 적용한다.  
- 이러한 변환을 통해 모델은 데이터의 표현을 재구성하여, 패턴이 더 쉽게 탐지되고, 구분되며, 예측될 수 있도록 한다.

#### 확률이론
- 상당히 많이 사용
- ML 모델을 다룰때 결국 real-world data를 다루는데 실제 데이터는 노이즈와 불확실성이 있고 불완전한 데이터임.
- 이런 불확실성을 다루는게 확률
- 확률론(probability theory)은 머신러닝에서 불확실성을 모델링하기 위한 틀을 제공  
- 실제 세계의 데이터는 종종 잡음이 많고, 불확실하며, 불완전하기 때문에, 모델은 입력과 출력 모두에서 나타나는 변동성과 모호성을 고려해야 함.  
- 확률분포(probability distributions), 조건부확률(conditional probability), 기댓값(expectation), 우도(likelihood) 같은 개념은 머신러닝 모델이 예측을 표현하고 데이터로부터 학습하는 방식에서 핵심적인 역할을 한다.

#### 정보이론
- 확률과 비슷하게 사용
- 몇가지 용어를 설명하는데 그 용어가 정보이론에 기반 되기도 함.
- 불확실성의 양을 측정할때 정보이론 많이 사용
- 정보이론(information theory)은 확률적 모델을 바탕으로, 정보량과 예측의 질을 정량적으로 측정하는 방법을 제공한다.  
- 엔트로피(entropy)와 같은 개념은 예측 집합에 존재하는 불확실성을 설명하고, 교차 엔트로피(cross-entropy)는 예측된 확률이 정답 결과와 얼마나 차이가 나는지를 측정하며, 이는 머신러닝에서 사용되는 많은 학습 목적 함수(training objectives)의 기초를 이룬다.

#### Calculus
- 특히 미분을 많이 이용
- 미적분학(calculus)은 모델의 매개변수 변화가 출력값과 전체 성능에 어떤 영향을 미치는지를 이해하는 데 필요한 도구를 제공한다.  
- 모델의 오차가 작은 조정에 따라 어떻게 달라지는지를 측정함으로써, 미적분학은 학습 알고리즘이 훈련 과정에서 매개변수를 체계적으로 업데이트하고 예측을 개선할 수 있게 해준다.

- 첫째, 이들은 미분(differentiation)과 기울기 계산(gradient computation)을 자동으로 처리하므로, 사용자가 복잡한 수학식을 직접 유도하고 구현해야 하는 부담을 덜어준다. 이로 인해 복잡한 모델도 더 정확하고 효율적으로 설계하고 학습시킬 수 있다.
- GPU 최적화 잘 되도록 딥러닝 패키지들이 기능 제공
- 또한 이들은 GPU나 TPU와 같은 하드웨어 가속을 활용할 수 있는 고도로 최적화된 구현을 제공하여, 그렇지 않으면 계산적으로 실행이 어려운 대규모 모델의 효율적인 학습을 가능하게 한다.
- 또한 딥러닝 패키지는 시각화 라이브러리, 배포 프레임워크, 그리고 다양한 사전학습 모델(pre-trained models)을 포함한 풍부한 지원 도구 생태계를 함께 제공한다.
- 이러한 자원들은 연구자와 실무자 모두가 실험 단계에서 실제 응용 단계로 더 빠르게 나아갈 수 있도록 도와준다.
